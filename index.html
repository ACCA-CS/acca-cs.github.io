ACCA Computer Science Division

Collegiate Coding Challenge

Overview

Welcome to our Full-Day Group Coding Challenge! This event is designed to bring together the brightest minds to tackle a series of tasks ranging from easy to hard difficulty levels. Teams will compete against each other to implement the most innovative, efficient, and effective solutions, with prizes for the top performers. Prepare to push your limits, collaborate with your teammates, and have fun!

Objective

Your team's objective is to select an open source project from the options below. For each project, you will find a collection of tasks of varying difficulty: Easy, Medium, and Hard. Your goal is to implement solutions that address the requirements of each task you choose to undertake.

Teams

  • Participants will be organized into teams of 3-5 members.
  • Collaboration is key. Make sure to leverage the strengths of each team member.

Rules

  1. Time Limit: The challenge will last from 9:30 A.M. to 3:00 P.M. All submissions must be made before the closing time announced on the day of the event.
  2. Submission: Each team must submit their solutions through the designated submission platform. Ensure your code is well-documented and includes instructions for setup and operation.
  3. Languages and Tools: You can use any programming language or development tools you are comfortable with, unless a task specifies otherwise.
  4. Fair Play: External assistance, including but not limited to, outsourcing work or using pre-existing solutions, is strictly prohibited. All work must be your own.

Judging Criteria

  • Features that teams implement have been catagorized into three types: Easy, worth one point each, Medium, worth three points each, and Hard, worth six points each. At the end of the competition, judges will evaluate the implementations and award points. The team with the most points will win first place, the team with the second most will win second place, and the team with the third most points will win third place.

Schedule

  • Opening Remarks: 9:00 AM
  • Coding Begins: 9:30 AM
  • Lunch Break: 12:00 PM - 1:00 PM
  • Coding Ends: 3:00 PM
  • Judging: 3:30 PM
  • Award Ceremony: 4:30 PM

Support

Throughout the day, mentors will be available to provide guidance and answer questions. If you need help, please don't hesitate to ask.

Ready? Let's go!

Prepare for an intense day of coding, learning, and networking. We encourage you to approach the challenge with an open mind and a spirit of collaboration. Good luck to all teams! Let the coding begin!

Projects

Rocket Chat

Easy:

  • Any Custom Status Message: Modify the pre-existing status message into a custom message, that would contain an emoji as a part of it.
  • Enlarge Font Size Option for Chat: Implement a feature that changes the default project font size in a chat room for better readability.
  • Customizable Chat Notification Sounds: Adding, thus, changing one of the inbuild notification sounds to a custom one.
  • Direct Message Read Receipts Toggle: Add an option for users to enable or disable read receipts specifically for some messages.
  • Theme Color Preview: create a logic where chat’s color is set based on chat’s name. i.e. Name is Defining the chat color.

Medium:

  • Advanced Chat Room Filters: Implement advanced filtering options for chat rooms, such as filtering by activity level, number of participants, or custom tags. This requires backend logic to sort/filter rooms and frontend updates for the UI.
  • User Mute/Unmute Notifications: Extend the mute/unmute feature to include optional notifications when a user is muted or unmuted in a chat room. This involves backend changes to handle the notification logic and frontend changes for displaying notifications.
  • Customizable Chat Room Layouts: Allow users to customize the layout of chat rooms (e.g., where the message input box is located, how messages are displayed). This requires significant frontend changes and possibly backend support for storing preferences.
  • Automated Chat Moderation Rules: Allow admins to set up automated rules for chat moderation (e.g., auto-mute for specific phrases). This requires backend logic for the rules engine and frontend changes for rule management.

Hard:

  • Advanced Analytics Dashboard for Chat Admins: Create an advanced analytics dashboard that provides detailed insights into chat room metrics (e.g., message volume, user engagement, sentiment analysis).
  • Language Learning Capabilities based on translations: There is a capability for translation of messages. But task is to implement the capability where user would be able to learn a new language, e.g. only partially translate messages, where translated text amount is reduced over time, likewise, create a mechanics for it. (I.e. initially, simple words are not translated, and with time more and more of them are not being translated, as time goes and number of non-translated text increases and, the difficulty – word complexity, would also increase, meaning more complicated words are not translated are also not being translated.)
  • End-to-End Encryption for Private Chats: Allow user to upload a custom key, e.g. custom RSA key and be able to use it instead for encryption.
  • Peer-to-Peer (P2P) File Sharing in Chat Rooms: Enable direct file sharing between users within a chat room without uploading to the server
  • Enhanced Security for Sensitive Messages: Implement a feature to mark message as “sensitive”, thus encrypting them in a way that only for a given user can read them, while others, might see just a scrambled text.
  • Privacy-Enhanced Messaging Mode: mode that would hide the message until it is being clicked or hovered over, and would be deleted after some set time.

FreeCiv

Easy:

  • Add a New Unit Type: Create a simple new unit type, which would be integrated into one of the cultures, or be generic for all (e.g., a unique scout with slightly altered stats from existing units)
  • Modify Unit Cost: Change the resource cost of producing a unit.
  • Introduce a New Building Bonus: Add a small bonus for an existing building (e.g., an additional happiness point for a library)
  • Adjust Technology Research Time: Slightly modify the time it takes to research a technology, aiming to balance gameplay. This requires changes to the technology definitions file.
  • Update Civilization Starting Units: Alter the starting units for a civilization, such as giving a civilization a unique unit at the start.

Medium:

  • Custom automation: Game has some automation features. Existing features: auto explore, auto work. New feature could be, automatic city development, or have auto defense/attack when enemy unit is discovered.
  • Implement a New Resource Type: Introduce a new resource type into the game, including its effects on city development or trade. This requires updates to the game's resource management system, including backend logic for resource effects and frontend updates for resource display.
  • New Civilization Attributes: Develop attributes or bonuses for existing civilizations, requiring more nuanced changes across game logic to implement and balance these new attributes properly.
  • Custom Victory Conditions Interface: Create an interface allowing players to select from various victory conditions before starting a game.

Hard:

  • Create a simple Map Editor for FreeCiv-web: Create a new “Map Editor” option next to “Start Game” and “Load Game”, which would allow to do some editing to maps.
  • Create a new Civilization: It need to have a different technology tree, and various units and buildings.
  • Dynamic Weather System: Implement a dynamic weather system affecting unit movement and combat, resource production, and map visibility. Create environmental effects, e.g. rain slows down units, ground tiles may reduce some unit parameters, or some cultures gain environmental benefits.
  • Advanced Modding Support: Develop a comprehensive modding framework allowing players to easily create and share mods that can alter game mechanics, add new content, and customize UI elements. This involves creating a robust API, documentation, and possibly a mod management interface.
  • Complete Overhaul of the Combat System: Introduce a new combat system with more variables (e.g., terrain, morale, weather). This complex challenge requires extensive backend modifications to the combat engine and frontend updates for displaying new combat information.
  • Implement Cultural Influence Mechanics: Introduce a system where cities or entire civilizations can exert cultural influence over others, potentially converting cities to the player's civilization over time. This feature involves complex game logic for calculating and applying cultural influence, UI updates to track influence progress, and adjustments to the diplomacy system.

AR-Toolbox

Easy:

  • Change the default model: Change the default first object that is chosen and placed.
  • Increase Text Size in UI: Adjust the default font size for all text elements in the UI to enhance readability.
  • Set Default Color for All objects: Altering a default color variable that is applied to new objects. (Even when another object is selected to be placed its default initial color need to be custom, without user setting it)
  • Increase the icon size: objects that user chooses are quite small, so they need to be increased in size.
  • Simplify the UI Dashboard: UI has a lot of information, to clean feature is to make it more minimalistic by removing some things from it.

Medium:

  • Add custom object in the menu (e.g., triangle): This involves creating a new menu item and possibly adding a new 3D model asset.
  • User Interface for Model Scaling: Create a slider interface that allows users to adjust the scale of selected 3D models more precisely than pinch gestures.
  • Model Transparency Control: Introduce a slider in the UI that lets users adjust the opacity of selected 3D models.
  • Object Grouping Feature: Allow users to group multiple objects together so they can be moved or modified all together.

Hard:

  • Object Interaction Scripts: Allow users to attach simple scripts to objects for predefined animations or interactions.
  • Environmental Reflection Mapping: Implement a feature that uses the device’s camera to create real-time reflections on shiny objects.
  • Real-time Multi-Layer Shadows: Create a more complex shadow system that allows for multiple light sources and corresponding shadows.
  • Advanced AR Measurement Tools: Develop a feature that allows users to measure distances and angles between multiple points or objects in 3D space, taking into account the perspective and scale of the AR environment.
  • Programmable AR Object Behaviors: Allow users to define behaviors for objects that can be triggered by specific events or conditions, such as an object that changes color when another object comes close.
  • Performance Tuning Settings: Provide advanced settings that allow users to optimize performance by adjusting the fidelity of textures or the complexity of shaders.
